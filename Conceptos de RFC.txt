¿Qué es HTTP?
El Protocolo de Transferencia de Hipertexto (HTTP) es un protocolo de aplicación sin estado que permite la comunicación en sistemas de información distribuidos y colaborativos, como la World Wide Web. Es la base de la comunicación en la web, permitiendo la transferencia de documentos hipermedia entre clientes y servidores. 
Arquitectura General de HTTP
    • Cliente y Servidor: HTTP sigue un modelo cliente-servidor. El cliente (por ejemplo, un navegador web) envía solicitudes, y el servidor responde con los recursos solicitados o mensajes de error.
    • Mensajes: La comunicación se realiza mediante mensajes HTTP, que pueden ser de dos tipos:
        ◦ Solicitud (Request): Enviada por el cliente para solicitar un recurso o realizar una acción.
        ◦ Respuesta (Response): Enviada por el servidor con el resultado de la solicitud.
    • Recursos: Son las entidades disponibles en el servidor, identificadas por un Identificador Uniforme de Recursos (URI). Un recurso puede ser un documento HTML, una imagen, un video, etc.
Métodos HTTP
Los métodos HTTP indican la acción que el cliente desea realizar sobre un recurso. Los más comunes son:
    • GET: Solicita la representación de un recurso específico. No debe usarse para operaciones que impliquen efectos secundarios.
    • POST: Envía datos al servidor para crear o modificar un recurso.
    • PUT: Reemplaza todas las representaciones actuales del recurso de destino con el contenido de la solicitud.
    • DELETE: Elimina el recurso especificado.
    • HEAD: Similar a GET, pero solo solicita los encabezados de la respuesta, sin el cuerpo.
    • OPTIONS: Describe las opciones de comunicación para el recurso de destino.
    • PATCH: Aplica modificaciones parciales a un recurso.
Códigos de Estado HTTP
Las respuestas del servidor incluyen códigos de estado que indican el resultado de la solicitud:
    • 1xx (Informativo): La solicitud fue recibida y el proceso continúa.
    • 2xx (Éxito): La solicitud se recibió, entendió y aceptó correctamente.
    • 3xx (Redirección): Se requiere una acción adicional para completar la solicitud.
    • 4xx (Error del Cliente): La solicitud contiene una sintaxis incorrecta o no puede procesarse.
    • 5xx (Error del Servidor): El servidor falló al cumplir con una solicitud válida.
Campos de Cabecera HTTP
Los mensajes HTTP contienen campos de cabecera que proporcionan información adicional sobre la solicitud o la respuesta. Algunos campos comunes incluyen:
    • Content-Type: Indica el tipo de medio del recurso.
    • Content-Length: Especifica la longitud del cuerpo en bytes.
    • Host: Especifica el nombre de dominio del servidor y, opcionalmente, el número de puerto.
    • User-Agent: Identifica el agente de usuario que realiza la solicitud.
    • Accept: Indica qué tipos de medios son aceptables para la respuesta.
Características Clave de HTTP
    • Sin Estado: Cada solicitud se realiza de forma independiente; el protocolo no mantiene información sobre solicitudes anteriores.
    • Extensible: HTTP permite la introducción de nuevos métodos, campos de cabecera y versiones sin interrumpir las comunicaciones existentes.
    • Conexiones Persistentes: Por defecto, HTTP/1.1 utiliza conexiones persistentes, lo que significa que se pueden enviar múltiples solicitudes y respuestas a través de una única conexión TCP, mejorando la eficiencia.
